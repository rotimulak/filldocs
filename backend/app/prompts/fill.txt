Вот список полей таблицы из документа Word в формате JSON. Каждое поле содержит row (номер строки), col (номер столбца), field (название поля), current_value (текущее значение ячейки):
{table_text}

Вот реквизиты компании в формате XML:
{requisites_xml}

Для каждого поля, которое нужно заполнить, верни объект с row, col и value.
Используй ТОЧНО те row и col, которые указаны в JSON выше.

Верни результат строго в формате JSON-массива (row и col бери из входного JSON, не придумывай свои):
[
  {{"row": <row из входного JSON>, "col": <col из входного JSON>, "value": "значение"}},
  ...
]

Правила:
- Используй row и col из входного JSON — не меняй их
- Если current_value уже заполнено — не перезаписывай
- Если для одной ячейки подходят несколько реквизитов (например "Паспортные данные" — серия, номер, дата, кем выдан), объедини их в одну строку через пробел или запятую в логичном порядке
- Ячейки с меткой "Фамилия, имя, отчество" или "ФИО" заполняй значением из реквизита "Директор" (или "Контактное лицо" / "Учредитель" если директора нет)
- Поле "Организационно-правовая форма" заполняй на основе наименования организации: ООО → "Общество с ограниченной ответственностью", АО → "Акционерное общество", ИП → "Индивидуальный предприниматель" и т.д.
- Если метка поля содержит "учредител" или "коллегиального исполнительного органа" — используй ИНН учредителя, а не ИНН компании
- Если документ заполняется для юридического лица (ООО/АО/ИП и т.д.), поля с пометкой "(для физического лица)" оставляй пустыми
- Пиши только само значение без пояснений
- Если для поля нет подходящего реквизита — не включай его в ответ
- Верни только JSON-массив, без дополнительного текста
