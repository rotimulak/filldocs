# Роли команды FillDocs

> **Навигация:** [← Главный индекс](../README.md)

---

## О проекте

FillDocs — веб-приложение для автоматического заполнения реквизитов компании в документах Word (.docx). Двухпанельный интерфейс: извлечение реквизитов из документа → заполнение шаблона.

## Определения ролей

| Роль | Зона ответственности |
|------|----------------------|
| [Архитектор](./architect.md) | Архитектура веб-приложения, выбор технологий, структура системы |
| [Системный аналитик](./system-analyst.md) | Требования, спецификации, пользовательские сценарии |
| [UX-дизайнер](./ux-designer.md) | Интерфейсы, UX двухпанельного приложения |
| [Fullstack-разработчик](./fullstack-developer.md) | Реализация API (FastAPI), Frontend (React + Vite) |
| [QA-инженер](./qa.md) | Unit-тесты, интеграционные тесты, покрытие кода |
| [Технический документалист](./techdoc.md) | Структура и актуальность документации |

---

## Принципы работы

Каждая роль имеет:
- Ключевые компетенции
- Принципы работы
- Типы артефактов
- Инструменты

Документы ролей описывают **как** выполнять работу, а не **что** делать в конкретном проекте.

## Использование ролей с Claude

Для активации роли в Claude Code используйте:

```
Выступи в роли [название роли] и [задача]
```

Пример:
```
Выступи в роли архитектора и проанализируй текущую структуру обработки .docx файлов
```
