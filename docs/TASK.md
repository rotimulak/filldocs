# Техническое задание: FillDocs

## Описание

Веб-приложение для автоматического заполнения реквизитов компании в документах Word (.docx).

## Левая панель: Реквизиты

### Функционал

1. **Drag & Drop зона** для загрузки документа с реквизитами
   - Принимает: `.doc`, `.docx`
   - Альтернатива: кнопка "Открыть файл"

2. **Распознавание реквизитов**
   - Отправка файла на backend
   - Парсинг таблицы с реквизитами
   - Извлечение полей в JSON

3. **Отображение результата**
   - JSON preview (красиво отформатированный)
   - Список распознанных полей с значениями
   - Возможность редактирования (опционально)

4. **Сохранение в localStorage**
   - Автоматическое сохранение после успешного распознавания
   - При загрузке страницы — восстановление из localStorage
   - Индикатор "✓ Реквизиты загружены" если есть сохранённые данные

### API endpoint

```
POST /api/extract-requisites
Content-Type: multipart/form-data

Request: file (binary)
Response: {
  "success": true,
  "requisites": {
    "company_name": "ООО Рога и Копыта",
    "inn": "1234567890",
    ...
  },
  "raw_fields": [
    {"label": "Наименование", "value": "ООО Рога и Копыта"},
    ...
  ]
}
```

## Правая панель: Заполнение

### Функционал

1. **Drag & Drop зона** для загрузки шаблона
   - Принимает: `.doc`, `.docx`
   - Альтернатива: кнопка "Открыть файл"
   - Неактивна пока нет реквизитов

2. **Индикатор загрузки**
   - Спиннер во время обработки
   - Текст: "Заполнение документа..."

3. **Отчёт о заполнении**
   - Список полей с статусом:
     - ✓ Заполнено (зелёный)
     - ✗ Не найдено (серый)
   - Счётчик: "Заполнено X из Y полей"

4. **Скачивание результата**
   - Кнопка "Скачать документ"
   - Имя файла: `{original_name}_filled.docx`

### API endpoint

```
POST /api/fill
Content-Type: multipart/form-data

Request:
  - file: binary (шаблон)
  - requisites: JSON string

Response: {
  "success": true,
  "filled_fields": [
    {"field": "company_name", "label": "Наименование", "filled": true, "count": 2},
    {"field": "inn", "label": "ИНН", "filled": true, "count": 1},
    {"field": "ogrn", "label": "ОГРН", "filled": false, "count": 0}
  ],
  "total_filled": 5,
  "total_fields": 12,
  "download_url": "/api/download/abc123.docx"
}
```

## Состояния UI

### Левая панель

| Состояние | Отображение |
|-----------|-------------|
| Пусто | Drag & Drop зона с иконкой |
| Загрузка | Спиннер + "Распознавание..." |
| Успех | JSON preview + "✓ Сохранено" |
| Ошибка | Красное сообщение об ошибке |
| Из localStorage | JSON preview + "Загружено из памяти" |

### Правая панель

| Состояние | Отображение |
|-----------|-------------|
| Нет реквизитов | Зона неактивна, текст "Сначала загрузите реквизиты" |
| Готово к загрузке | Drag & Drop зона активна |
| Загрузка | Спиннер + "Заполнение документа..." |
| Успех | Отчёт + кнопка "Скачать" |
| Ошибка | Красное сообщение |

## Технический стек

### Frontend
- React 18 + TypeScript
- Tailwind CSS
- Vite

### Backend
- FastAPI (Python)
- python-docx

### Хранение
- localStorage для реквизитов
- Временные файлы на сервере для результатов
