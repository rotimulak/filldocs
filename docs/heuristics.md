# Эвристики поиска и заполнения реквизитов

## Правила нахождения блока реквизитов

1. **Реквизиты находятся в таблице**
   - Документ должен содержать таблицу с реквизитами
   - Если таблиц нет — реквизиты не найдены

2. **Структура таблицы: 2 или 3 колонки**
   - 2 колонки: `[Название поля | Значение]`
   - 3 колонки: `[№ п/п | Название поля | Значение]`

3. **Блок реквизитов — одна таблица**
   - Все реквизиты компании находятся в рамках одной таблицы
   - Если ключевые слова найдены в других таблицах — это ложные срабатывания, игнорировать

4. **Идентификация таблицы реквизитов**
   - Таблица считается "блоком реквизитов" если содержит ≥3 ключевых слов из маппинга
   - Приоритет: таблица с наибольшим количеством совпадений

---

## Правила заполнения

5. **Ячейка для заполнения должна быть пустой**
   - Допустимо: пробелы, переносы строк, непечатаемые символы
   - Проверка: `cell.text.strip() == ""`

6. **Перезапись данных запрещена**
   - Если ячейка содержит видимый текст — это ошибка
   - Вернуть предупреждение: "Ячейка уже заполнена: {label}"

7. **Стиль текста берётся из таблицы**
   - Шрифт, размер, выравнивание — копировать из соседних ячеек
   - Если не определено — использовать стиль по умолчанию документа

---

## Правила обработки составных полей

8. **Несколько ключевиков в одной ячейке**
   - Пример: ячейка "ИНН / КПП" содержит два ключевика
   - Действие: вставить значения через пробел: `"1234567890 123456789"`
   - Порядок: в том же порядке, как указаны ключевики в ячейке

9. **Дедупликация составных полей**
   - Если "ИНН / КПП" уже заполнено, а ниже есть отдельная ячейка "КПП"
   - Действие: пропустить отдельную ячейку (уже вставлено в составное поле)
   - Логика: отслеживать использованные ключи

---

## Порядок обработки

```
1. Найти все таблицы в документе
2. Для каждой таблицы подсчитать совпадения с ключевыми словами
3. Выбрать таблицу с максимальным числом совпадений (≥3)
4. Для каждой строки выбранной таблицы:
   a. Извлечь текст метки (col 1 или col 2)
   b. Найти все ключевики в метке
   c. Проверить что ячейка значения пуста
   d. Если несколько ключевиков — объединить значения
   e. Отметить использованные ключи
   f. Записать значение с сохранением стиля
5. Вернуть отчёт: заполнено / пропущено / ошибки
```

---

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Нет таблиц в документе | Ошибка: "Таблицы не найдены" |
| Нет таблицы с ≥3 ключевиками | Ошибка: "Блок реквизитов не найден" |
| Ячейка уже заполнена | Предупреждение, пропустить поле |
| Ключевик не найден в реквизитах | Пропустить (не ошибка) |
| Составное поле частично заполнено | Заполнить только пустую часть |

## Базовая валидация реквизитов

### Правила валидации при извлечении

| Поле | Правило | Действие при ошибке |
|------|---------|---------------------|
| ИНН | 10 цифр (ЮЛ) или 12 цифр (ИП/ФЛ) | Предупреждение, сохранить как есть |
| КПП | 9 цифр | Предупреждение, сохранить как есть |
| ОГРН | 13 цифр (ЮЛ) или 15 цифр (ИП) | Предупреждение, сохранить как есть |
| БИК | 9 цифр | Предупреждение, сохранить как есть |
| Расчётный счёт | 20 цифр | Предупреждение, сохранить как есть |
| Корр. счёт | 20 цифр | Предупреждение, сохранить как есть |
| Email | Содержит @ и домен | Предупреждение, сохранить как есть |
| Телефон | Начинается с + или 8, содержит цифры | Предупреждение, сохранить как есть |

### Принцип валидации

- **Мягкая валидация**: предупреждаем, но не блокируем
- Данные сохраняются "как есть" даже при ошибке формата
- Предупреждения выводятся в отчёт извлечения
- Пользователь может исправить вручную или проигнорировать

### Реализация

```python
VALIDATORS = {
    "inn": lambda v: len(re.sub(r'\D', '', v)) in [10, 12],
    "kpp": lambda v: len(re.sub(r'\D', '', v)) == 9,
    "ogrn": lambda v: len(re.sub(r'\D', '', v)) in [13, 15],
    "bik": lambda v: len(re.sub(r'\D', '', v)) == 9,
    "account": lambda v: len(re.sub(r'\D', '', v)) == 20,
    "corr_account": lambda v: len(re.sub(r'\D', '', v)) == 20,
    "email": lambda v: "@" in v and "." in v.split("@")[-1],
    "phone": lambda v: bool(re.match(r'^[\+8][\d\s\-\(\)]+$', v)),
}

def validate_requisites(data: dict) -> list[str]:
    """Возвращает список предупреждений"""
    warnings = []
    for key, value in data.items():
        if key in VALIDATORS and value:
            if not VALIDATORS[key](value):
                warnings.append(f"Некорректный формат {key}: {value}")
    return warnings
```

### Нормализация данных

При извлечении применяется базовая нормализация:

| Поле | Нормализация |
|------|--------------|
| ИНН, КПП, ОГРН, БИК | Удалить все кроме цифр |
| Счета | Удалить пробелы и дефисы |
| Телефон | Оставить как есть (формат может варьироваться) |
| Email | Привести к нижнему регистру, trim |
| Остальные | Только trim пробелов |
